Civ3 Cross-Platform Editor Guide

Welcome to the guide to the Civ3 cross platform editor. This site serves as the base to the online help system. For questions, please visit the forum thread at here.

System Requirements

The only two requirements are a 1024x768 or higher resolution monitor (this is also the minimum for Civ3), and a system that can support Java 1.5. Such systems include:

  • Windows 98/ME/2000 or later
  • Mac OSX 10.4 (Tiger) or later
  • Recent versions of Linux (exact requirement unknown)

It is recommended that you use Sun's Java versions. Other versions may work, but will not be supported.

Setup

Once you have downloaded the file from here, unzip the .zip file. Then simply double-click on the .jar file, and the editor will start up. You will be asked to select your Civilization III install folder, so that the editor will be able to load images and set your default file open/save folder to wherever your default Conquests scenario folder is. You will only be asked to do this the first time you run the program.

Limitations

Currently, the editor only supports BIQ files that:

  • Are version 12.08 or 12.06 (corresponding to Conquests 1.22)
  • Have custom rules

Overview

Upon starting the editor, you will see a screen that looks similar to the following:

The basic functions - Open and Save - are straightforward, as one would expect. However, there are concepts to be expounded upon:

  • Input From SAV: The editor includes support from extracting BIQ files from SAV files that have them embedded. Note the SAV must be uncompressed.
  • Safety Level: The editor supports a number of different "safety levels". A lower safety level enables more elements to be modified, beyond what the classic editor allows.
  • BIQC tab: This tab allows the comparison of multiple BIQ files, automatically and in a format that is (somewhat) easy for humans to read

Gotchas

Beyond the requirements and limitations, there are a few other potential gotchas.

Opening multiple files: The editor is not guaranteed to work properly with multiple files open. It may work correctly, but it is not recommended that you open multiple files at once, unless you are using the BIQC functionality, which does correctly support multiple files.

Incomplete functionality: Some areas may not include complete functionality. Details about this can be found on the specialized instructions for the respective tab.

Red text: Red text indicates that the current value does not fall within the safety level's requirements. For example, if you tried to give a unit 21 hit points with the default safety level, you would notice that the hit point value (21) was in red text. The editor does not currently enforce the safety level's limits, but the red text serves as a warning that you are violating the bounds of the tab's current safety level

Invalid values: It is possible to enter too large a value for a field for it to be processed - for example, the value 5 billion is likely to cause this anywhere.
